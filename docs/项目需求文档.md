# Email-DB 项目需求开发文档

## 1. 项目概述

### 1.1 项目名称
email-cf-d1 - 基于 Cloudflare 的邮箱账号管理系统

### 1.2 项目背景
开发一个轻量级的邮箱账号密码管理系统，用于集中存储和管理多个邮箱账号信息，便于运维、测试或个人使用场景。

### 1.3 项目目标
- 提供简洁的 Web UI 界面进行增删改查操作
- 提供 API 接口支持批量导入邮箱数据
- 使用 Cloudflare 全栈技术实现零成本部署
- 通过 GitHub 实现自动化 CI/CD 部署

## 2. 技术栈

### 2.1 前端技术
- **框架**: 纯 HTML/CSS/JavaScript (无构建依赖)
- **样式**: 原生 CSS，响应式设计
- **交互**: Vanilla JavaScript, Fetch API

### 2.2 后端技术
- **运行时**: Cloudflare Workers
- **框架**: Cloudflare Pages Functions
- **语言**: TypeScript

### 2.3 数据库
- **类型**: Cloudflare D1 (SQLite)
- **ORM**: 原生 D1 Binding

### 2.4 部署平台
- **托管**: Cloudflare Pages
- **CI/CD**: GitHub 集成自动部署
- **域名**: Cloudflare 提供的默认域名或自定义域名

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 邮箱管理 (CRUD)
**查询功能**
- 显示所有邮箱账号列表
- 字段：ID、邮箱地址、密码、备注、创建时间
- 按创建时间倒序排列

**添加功能**
- 输入字段：邮箱地址（必填）、密码（必填）、备注（可选）
- 前端表单验证
- 成功后刷新列表

**编辑功能**
- 点击编辑按钮弹出表单
- 预填充现有数据
- 支持修改所有字段
- 更新时间自动记录

**删除功能**
- 删除前二次确认
- 删除后刷新列表

#### 3.1.2 批量上传接口
**接口特性**
- 仅通过 API 调用（非 UI 操作）
- 需要 API Key 认证
- **支持单个上传和批量上传两种格式**
- **自动跳过重复邮箱，不保存**
- 返回成功/失败/跳过统计及详细信息

**单个上传格式**
```json
POST /api/upload
Headers:
  Authorization: Bearer <API_KEY>
  Content-Type: application/json

Body:
{
  "email": "test@example.com",
  "password": "password123",
  "description": "备注信息"
}
```

**批量上传格式**
```json
POST /api/upload
Headers:
  Authorization: Bearer <API_KEY>
  Content-Type: application/json

Body:
{
  "emails": [
    {
      "email": "test1@example.com",
      "password": "password1",
      "description": "备注1"
    },
    {
      "email": "test2@example.com",
      "password": "password2",
      "description": "备注2"
    }
  ]
}
```

**响应格式**
```json
{
  "success": true,
  "data": {
    "total": 5,
    "success": 3,
    "skipped": 2,
    "failed": 0,
    "skippedItems": [  // 被跳过的重复邮箱
      "duplicate@example.com"
    ],
    "failedItems": [  // 仅当有失败项时返回
      {
        "email": "invalid@example.com",
        "reason": "Database error"
      }
    ]
  }
}
```

### 3.2 非功能需求

#### 3.2.1 性能要求
- 页面加载时间 < 2 秒
- API 响应时间 < 500ms
- 支持至少 1000 条记录流畅操作

#### 3.2.2 安全要求
- API Key 通过环境变量配置
- HTTPS 强制加密传输
- 防止 SQL 注入（使用参数化查询）
- 防止 XSS 攻击（前端转义）

#### 3.2.3 可用性要求
- 响应式设计，支持移动端访问
- 界面简洁直观，无需培训即可使用
- 操作反馈及时（成功/失败提示）

## 4. 数据库设计

### 4.1 数据表结构

#### emails 表
```sql
CREATE TABLE emails (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  email TEXT NOT NULL,
  password TEXT NOT NULL,
  description TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**字段说明**:
- `id`: 主键，自增
- `email`: 邮箱地址，不可为空
- `password`: 密码，不可为空（明文存储，按需可升级加密）
- `description`: 备注信息，可为空
- `created_at`: 创建时间，自动生成
- `updated_at`: 更新时间，修改时自动更新

**索引**:
```sql
CREATE INDEX idx_email ON emails(email);
```

## 5. API 接口设计

### 5.1 RESTful API 规范

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| GET | /api/emails | 获取所有邮箱 | 否 |
| POST | /api/emails | 创建邮箱 | 否 |
| PUT | /api/emails/:id | 更新邮箱 | 否 |
| DELETE | /api/emails/:id | 删除邮箱 | 否 |
| POST | /api/upload | 批量上传 | **是** |

### 5.2 请求/响应格式

#### 统一响应结构
```json
{
  "success": true/false,
  "data": {},
  "error": "错误信息"
}
```

#### GET /api/emails
**响应示例**:
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "email": "test@example.com",
      "password": "password123",
      "description": "测试账号",
      "created_at": "2024-01-01 12:00:00",
      "updated_at": "2024-01-01 12:00:00"
    }
  ]
}
```

#### POST /api/emails
**请求体**:
```json
{
  "email": "new@example.com",
  "password": "newpass",
  "description": "新账号"
}
```

**响应**:
```json
{
  "success": true,
  "data": {
    "id": 2
  }
}
```

#### PUT /api/emails/:id
**请求体**:
```json
{
  "email": "updated@example.com",
  "password": "updatedpass",
  "description": "更新后的备注"
}
```

#### DELETE /api/emails/:id
**响应**:
```json
{
  "success": true
}
```

## 7. 开发流程

### 7.1 环境准备
1. 安装 Node.js (v18+)
2. 安装 Wrangler CLI: `npm install -g wrangler`
3. 登录 Cloudflare: `wrangler login`
4. 克隆项目代码

### 7.2 本地开发
```bash
# 1. 安装依赖
npm install

# 2. 创建 D1 数据库
npm run db:create

# 3. 初始化本地数据库
npm run db:local

# 4. 配置 .dev.vars 文件
# API_KEY=your-secret-key

# 5. 启动开发服务器
npm run dev
```

### 7.3 测试验证
- 访问 http://localhost:8788
- 测试增删改查功能
- 使用 cURL 或 Postman 测试上传接口

### 7.4 部署上线
```bash
# 方式 1: 命令行部署
npm run db:init        # 初始化生产数据库
npm run deploy         # 部署到 Cloudflare Pages

# 方式 2: GitHub 自动部署
git push origin main   # 推送后自动触发部署
```

## 9. 安全考虑

### 9.1 当前安全措施
1. **API Key 认证**: 上传接口需要 Bearer Token
2. **HTTPS 加密**: Cloudflare 自动提供 SSL
3. **SQL 注入防护**: 使用参数化查询
4. **XSS 防护**: 前端 HTML 转义

### 9.2 可选增强措施
1. **密码加密**: 使用 bcrypt 或 AES 加密存储
2. **访问控制**: 添加 HTTP Basic Auth 保护整个应用
3. **IP 白名单**: 限制特定 IP 访问
4. **审计日志**: 记录所有操作日志
5. **速率限制**: 防止暴力破解和滥用

## 11. 成本估算

### 11.1 Cloudflare 免费额度
- **Pages**: 500 次构建/月
- **Workers**: 100,000 次请求/天
- **D1**: 5GB 存储，500 万行读取/天

### 11.2 预期使用量
- 小规模使用（<1000 条记录，<1000 次/天访问）完全免费
- 超出免费额度后，按量付费（价格低廉）

## 12. 技术风险与对策

### 12.1 潜在风险
1. **数据丢失**: D1 数据库故障
   - 对策: 定期导出备份
2. **API 滥用**: 未授权访问上传接口
   - 对策: 强 API Key + 速率限制
3. **密码泄露**: 明文存储密码
   - 对策: 实施加密存储方案

### 12.2 降级方案
如 Cloudflare 服务不可用，可迁移至：
- Vercel + Neon (PostgreSQL)
- Netlify + Supabase
- 传统服务器 + MySQL/PostgreSQL
