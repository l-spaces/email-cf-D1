# email-cf-d1 部署指南

本文档介绍如何将 email-cf-d1 部署到 Cloudflare Pages 生产环境。

## 部署方式对比

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| GitHub 自动部署 | 自动化，推送即部署 | 需配置 GitHub 集成 | **推荐** - 团队协作 |
| 命令行部署 | 快速直接 | 手动操作 | 个人项目、快速测试 |

## 方式一：GitHub 自动部署（推荐）

### 步骤 1: 准备 GitHub 仓库

#### 1.1 创建 GitHub 仓库
```bash
# 在 GitHub 上创建新仓库，然后本地关联
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/your-username/email-cf-d1.git
git push -u origin main
```

#### 1.2 确保必要文件已提交
```bash
git status

# 应该包含：
# ✅ public/
# ✅ functions/
# ✅ schema.sql
# ✅ wrangler.toml
# ✅ package.json
# ❌ .dev.vars (不要提交)
```

### 步骤 2: 创建 Cloudflare Pages 项目

#### 2.1 登录 Cloudflare Dashboard
访问 https://dash.cloudflare.com/

#### 2.2 创建 Pages 项目
1. 进入 **Workers & Pages**
2. 点击 **Create application**
3. 选择 **Pages** 标签
4. 点击 **Connect to Git**

#### 2.3 连接 GitHub
1. 选择 **GitHub**
2. 授权 Cloudflare 访问你的 GitHub 账号
3. 选择 `email-cf-d1` 仓库（或你的仓库名）
4. 点击 **Begin setup**

#### 2.4 配置构建设置
在配置页面填写：

- **Project name**: `email-cf-d1`（或自定义）
- **Production branch**: `main`
- **Build command**: 留空
- **Build output directory**: `public`
- **Root directory**: `/`

点击 **Save and Deploy**

### 步骤 3: 创建和配置 D1 数据库

#### 3.1 创建生产数据库
回到本地终端执行：
```bash
wrangler d1 create email-database
```

记录返回的 `database_id`，例如：
```
database_id = "12345678-abcd-1234-5678-abcdef123456"
```

#### 3.2 更新 wrangler.toml
将 `database_id` 填入 `wrangler.toml`：
```toml
[[d1_databases]]
binding = "DB"
database_name = "email-database"
database_id = "12345678-abcd-1234-5678-abcdef123456"  # 替换为你的 ID
```

提交更改：
```bash
git add wrangler.toml
git commit -m "Update D1 database ID"
git push
```

#### 3.3 初始化生产数据库
```bash
wrangler d1 execute email-database --file=./schema.sql
```

执行成功后显示：
```
🌀 Executing on remote database email-database (12345678-abcd-1234-5678-abcdef123456):
✅ Success
```

### 步骤 4: 绑定 D1 到 Pages 项目

#### 4.1 进入项目设置
1. 在 Cloudflare Dashboard 中打开你的 Pages 项目
2. 进入 **Settings** 标签
3. 找到 **Functions** 部分

#### 4.2 添加 D1 Binding
1. 滚动到 **D1 database bindings**
2. 点击 **Add binding**
3. 填写：
   - **Variable name**: `DB`
   - **D1 database**: 选择 `email-database`
4. 点击 **Save**

### 步骤 5: 配置环境变量

#### 5.1 添加 API_KEY
1. 在 Settings 页面找到 **Environment variables**
2. 选择 **Production** 环境
3. 点击 **Add variable**
4. 填写：
   - **Variable name**: `API_KEY`
   - **Value**: 你的密钥（例如：`prod-secret-key-2024`）
5. 点击 **Save**

#### 5.2 生成强密钥（可选）
```bash
# 使用 Node.js 生成随机密钥
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### 步骤 6: 触发重新部署

方式 1: 推送代码
```bash
git commit --allow-empty -m "Trigger redeploy"
git push
```

方式 2: 手动触发
1. 在 Pages 项目页面
2. 进入 **Deployments** 标签
3. 点击 **Retry deployment**

### 步骤 7: 验证部署

#### 7.1 访问部署 URL
部署成功后会显示 URL，例如：
```
https://email-cf-d1.pages.dev
```

#### 7.2 测试功能
1. 打开页面，检查 UI 是否正常
2. 尝试添加一条邮箱记录
3. 测试编辑和删除功能

#### 7.3 测试 API 接口
```bash
# 查询接口
curl https://email-cf-d1.pages.dev/api/emails

# 单个上传接口（替换 YOUR_API_KEY）
curl -X POST https://email-cf-d1.pages.dev/api/upload \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "testpass123",
    "description": "测试账号"
  }'

# 批量上传接口
curl -X POST https://email-cf-d1.pages.dev/api/upload \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "emails": [
      {
        "email": "test@example.com",
        "password": "testpass123",
        "description": "测试账号"
      }
    ]
  }'
```

### 步骤 8: 配置自定义域名（可选）

#### 8.1 添加域名
1. 在 Pages 项目中点击 **Custom domains**
2. 点击 **Set up a custom domain**
3. 输入你的域名（例如：`email.yourdomain.com`）
4. 点击 **Continue**

#### 8.2 配置 DNS
根据提示添加 CNAME 记录：
```
CNAME email email-cf-d1.pages.dev
```

#### 8.3 等待 SSL 证书
Cloudflare 会自动颁发 SSL 证书，通常 1-5 分钟内完成。

## 方式二：命令行部署

### 前提条件
```bash
# 确保已登录
wrangler whoami

# 如未登录
wrangler login
```

### 步骤 1: 创建 D1 数据库
```bash
wrangler d1 create email-database
```

### 步骤 2: 更新 wrangler.toml
填入返回的 `database_id`。

### 步骤 3: 初始化数据库
```bash
wrangler d1 execute email-database --file=./schema.sql
```

### 步骤 4: 部署项目
```bash

```

或者直接使用：
```bash
wrangler pages deploy public
```

首次部署会提示创建项目：
```
✔ Enter the name of your new project: … email-cf-d1
✔ Enter the production branch name: … main
```

### 步骤 5: 配置环境变量
```bash
wrangler pages secret put API_KEY

```

在 Cloudflare Dashboard 手动绑定 D1（参考方式一步骤 4）

### 步骤 7: 验证部署
访问输出的 URL，测试功能。



## 常见问题

### Q1: 部署后 API 返回 500 错误

**原因**: D1 未绑定或数据库未初始化

**解决**:
```bash
# 检查 binding
wrangler pages project list

# 确保执行了初始化
wrangler d1 execute email-database --file=./schema.sql
```

### Q2: 上传接口返回 401 Unauthorized

**原因**: API_KEY 未配置或不匹配

**解决**:
```bash
# 检查环境变量
wrangler pages secret list

# 重新设置
wrangler pages secret put API_KEY
```

### Q3: 页面显示 404

**原因**: Build output directory 配置错误

**解决**:
确保 `wrangler.toml` 和 Dashboard 中都设置为 `public`。

### Q4: 数据库操作超时

**原因**: D1 免费版限制或网络问题

**解决**:
- 减少单次查询的数据量
- 添加索引优化查询
- 检查 Cloudflare 状态页

### Q5: 自动部署未触发

**原因**: GitHub 集成配置问题

**解决**:
1. Settings > Builds & deployments
2. 检查 **Build configuration** 是否正确
3. 检查 GitHub webhook 是否启用
4. 重新连接 GitHub 仓库

### Q6: 如何查看错误日志？

**方法 1**: Dashboard
1. Workers & Pages > 选择项目
2. **Logs** 标签
3. 选择时间范围

**方法 2**: 命令行
```bash
wrangler pages deployment tail
```

### Q7: 如何限制访问？

**方法 1**: Cloudflare Access（推荐）
1. 在 Dashboard 启用 Cloudflare Access
2. 创建访问策略（IP 白名单、Email 验证等）

**方法 2**: 自定义认证
参考开发指南的 HTTP Basic Auth 实现。

### Q8: 如何监控使用量？

**查看配额**:
1. Workers & Pages > 选择项目
2. **Metrics** 标签

**免费额度**:
- Pages: 500 次构建/月
- Workers: 100,000 次请求/天
- D1: 5GB 存储，500 万行读取/天

### Q9: 如何删除部署？

**删除项目**:
```bash
wrangler pages project delete email-cf-d1
```

**删除数据库**:
```bash
wrangler d1 delete email-database
```

⚠️ **警告**: 数据将永久丢失，请先备份！

## 性能优化

### 1. 启用 Cloudflare 缓存
在 `functions/_middleware.ts` 添加：
```typescript
export const onRequest: PagesFunction = async (context) => {
  const response = await context.next();
  
  // 静态资源缓存 1 小时
  if (context.request.url.match(/\.(js|css|png|jpg|svg)$/)) {
    response.headers.set('Cache-Control', 'public, max-age=3600');
  }
  
  return response;
};
```

### 2. 优化 D1 查询
```typescript
// 添加索引
CREATE INDEX idx_email ON emails(email);
CREATE INDEX idx_created_at ON emails(created_at);

// 分页查询
SELECT * FROM emails ORDER BY created_at DESC LIMIT 20 OFFSET 0;
```

### 3. 启用 Gzip 压缩
Cloudflare 自动提供，无需配置。

## 安全加固

### 1. 限制请求频率
使用 Cloudflare Rate Limiting（付费功能）或自定义实现：

```typescript
// functions/_middleware.ts
const requestCounts = new Map();

export const onRequest: PagesFunction = async (context) => {
  const ip = context.request.headers.get('CF-Connecting-IP');
  const count = requestCounts.get(ip) || 0;
  
  if (count > 100) {  // 100 次/分钟
    return new Response('Too Many Requests', { status: 429 });
  }
  
  requestCounts.set(ip, count + 1);
  setTimeout(() => requestCounts.delete(ip), 60000);
  
  return context.next();
};
```

### 2. 启用 WAF 规则
在 Cloudflare Dashboard:
1. Security > WAF
2. 启用 OWASP 规则集

### 3. 配置 CORS（如需要）
```typescript
export const onRequest: PagesFunction = async (context) => {
  const response = await context.next();
  
  response.headers.set('Access-Control-Allow-Origin', 'https://yourdomain.com');
  response.headers.set('Access-Control-Allow-Methods', 'GET, POST, PUT, DELETE');
  
  return response;
};
```
