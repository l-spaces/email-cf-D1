## 环境搭建

### 前置要求
- **Node.js**: >= 18.0.0
- **npm**: >= 9.0.0
- **Git**: 任意版本
- **Cloudflare 账号**: 免费账号即可

### 安装步骤

#### 1. 安装 Wrangler CLI
```bash
npm install -g wrangler
```

#### 2. 登录 Cloudflare
```bash
wrangler login
```
浏览器会打开授权页面，完成登录后返回终端。

#### 3. 克隆项目
```bash
git clone <your-repo-url>
cd Email-DB
```

#### 4. 安装项目依赖
```bash
npm install
```

#### 5. 创建 D1 数据库
```bash
# 创建数据库
wrangler d1 create email-database
```

执行后会返回类似信息：
```
✅ Successfully created DB 'email-database'

[[d1_databases]]
binding = "DB"
database_name = "email-database"
database_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
```

**复制 `database_id` 并更新 `wrangler.toml` 文件**:
```toml
[[d1_databases]]
binding = "DB"
database_name = "email-database"
database_id = "你的-database-id"  # 替换这里
```

#### 6. 初始化本地数据库
```bash
npm run db:local
```

#### 7. 配置环境变量
创建 `.dev.vars` 文件（已在 `.gitignore` 中）:
```bash
API_KEY=your-secret-api-key-change-this
```

⚠️ **重要**: 不要将 `.dev.vars` 提交到 Git！

#### 8. 启动开发服务器
```bash
npm run dev
```

访问 http://localhost:8788

## 核心技术

### 1. Cloudflare Pages Functions

Pages Functions 是基于 Cloudflare Workers 的 serverless 函数。

**文件路由规则**:
- `functions/api/emails.ts` → `/api/emails`
- `functions/api/emails/[id].ts` → `/api/emails/:id`
- `functions/_middleware.ts` → 所有请求的中间件

**导出的函数名对应 HTTP 方法**:
```typescript
// GET /api/emails
export const onRequestGet = async (context) => { ... }

// POST /api/emails
export const onRequestPost = async (context) => { ... }

// PUT /api/emails/:id
export const onRequestPut = async (context) => { ... }

// DELETE /api/emails/:id
export const onRequestDelete = async (context) => { ... }

// 所有方法
export const onRequest = async (context) => { ... }
```

### 2. Cloudflare D1

D1 是 Cloudflare 的 SQLite 数据库。

**访问方式**:
```typescript
interface Env {
  DB: D1Database;  // 绑定的数据库
}

export const onRequestGet: PagesFunction<Env> = async (context) => {
  const db = context.env.DB;
  
  // 查询
  const { results } = await db.prepare('SELECT * FROM emails').all();
  
  // 插入
  const result = await db.prepare(
    'INSERT INTO emails (email, password) VALUES (?, ?)'
  ).bind('test@example.com', 'password').run();
  
  // 更新
  await db.prepare('UPDATE emails SET email = ? WHERE id = ?')
    .bind('new@example.com', 1).run();
  
  // 删除
  await db.prepare('DELETE FROM emails WHERE id = ?')
    .bind(1).run();
};
```

**参数化查询**（防止 SQL 注入）:
```typescript
// ✅ 正确 - 使用 bind()
db.prepare('SELECT * FROM emails WHERE id = ?').bind(id)

// ❌ 错误 - 字符串拼接
db.prepare(`SELECT * FROM emails WHERE id = ${id}`)
```

### 3. 中间件认证

`functions/_middleware.ts` 会拦截所有请求：

```typescript
export const onRequest: PagesFunction<Env> = async (context) => {
  const url = new URL(context.request.url);

  // 仅对 /api/upload 验证
  if (url.pathname === '/api/upload') {
    const authHeader = context.request.headers.get('Authorization');
    const token = authHeader?.replace('Bearer ', '');

    if (!token || token !== context.env.API_KEY) {
      return new Response(JSON.stringify({ error: 'Unauthorized' }), {
        status: 401,
        headers: { 'Content-Type': 'application/json' }
      });
    }
  }

  // 继续处理请求
  return context.next();
};
```

使用 Vanilla JavaScript + Fetch API，无框架依赖。

**核心函数**:

```typescript
  // 使用 context.env 访问环境变量

  // 使用 Response.json() 返回 JSON

};
```

**JavaScript**:

```javascript
  }

// 使用 const/let，避免 var
```

### 2. 错误处理

**后端**:
```typescript
export const onRequestPost: PagesFunction<Env> = async (context) => {
  try {
    const body = await context.request.json();
    
    // 验证输入
    if (!body.email || !body.password) {
      return Response.json(
        { success: false, error: 'Email and password are required' },
        { status: 400 }
      );
    }
    
    // 执行操作
    const result = await context.env.DB.prepare('...').run();
    
    return Response.json({ success: true, data: result });
  } catch (error) {
    return Response.json(
      { success: false, error: 'Internal server error' },
      { status: 500 }
    );
  }
};
```

**前端**:

## 常见问题

### Q1: 如何修改数据库结构？

**步骤**:
1. 修改 `schema.sql`
2. 执行本地更新: `npm run db:local`
3. 执行生产更新: `npm run db:init`

⚠️ **注意**: D1 不支持自动迁移，需要手动编写 ALTER TABLE 语句。

### Q2: 如何调试 API 接口？

**方法 1**: 使用浏览器开发者工具
- 打开 Network 标签查看请求/响应

**方法 2**: 使用 cURL
```bash
# 查询所有邮箱
curl http://localhost:8788/api/emails

# 添加邮箱
curl -X POST http://localhost:8788/api/emails \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"pass123"}'

# 批量上传（需要 API Key）
curl -X POST http://localhost:8788/api/upload \
  -H "Authorization: Bearer your-api-key" \
  -H "Content-Type: application/json" \
  -d '{"emails":[{"email":"test@example.com","password":"pass123"}]}'
```

**方法 3**: 使用 Postman 或 Insomnia

### Q5: 如何添加新的 API 接口？

**示例**: 添加搜索接口

1. 创建文件 `functions/api/search.ts`:
```typescript
interface Env {
  DB: D1Database;
}

export const onRequestGet: PagesFunction<Env> = async (context) => {
  const url = new URL(context.request.url);
  const keyword = url.searchParams.get('q');

  const { results } = await context.env.DB.prepare(
    'SELECT * FROM emails WHERE email LIKE ?'
  ).bind(`%${keyword}%`).all();

  return Response.json({ success: true, data: results });
};
```

2. 访问 `/api/search?q=test`

### Q6: 如何实现分页？

**后端**:
```typescript
export const onRequestGet: PagesFunction<Env> = async (context) => {
  const url = new URL(context.request.url);
  const page = parseInt(url.searchParams.get('page') || '1');
  const pageSize = parseInt(url.searchParams.get('pageSize') || '20');
  const offset = (page - 1) * pageSize;

  const { results } = await context.env.DB.prepare(
    'SELECT * FROM emails LIMIT ? OFFSET ?'
  ).bind(pageSize, offset).all();

  return Response.json({ success: true, data: results, page, pageSize });
};
```

**前端**:
```javascript
async function loadEmails(page = 1, pageSize = 20) {
  const response = await fetch(`/api/emails?page=${page}&pageSize=${pageSize}`);
  return response.json();
}
```

```typescript
export const onRequestGet: PagesFunction<Env> = async (context) => {
  const { results } = await context.env.DB.prepare('SELECT * FROM emails').all();
  
  const csv = [
    'ID,Email,Password,Description,Created At',
```

### Q9: 如何添加用户认证？

**简单方案**: HTTP Basic Auth

在 `functions/_middleware.ts` 添加：
```typescript
export const onRequest: PagesFunction<Env> = async (context) => {
  const authHeader = context.request.headers.get('Authorization');
  
  if (!authHeader || !authHeader.startsWith('Basic ')) {
    return new Response('Unauthorized', {
      status: 401,
      headers: { 'WWW-Authenticate': 'Basic realm="Email DB"' }
    });
  }
  
  const base64Credentials = authHeader.split(' ')[1];
  const credentials = atob(base64Credentials);
  const [username, password] = credentials.split(':');
  
  if (username !== 'admin' || password !== context.env.ADMIN_PASSWORD) {
    return new Response('Unauthorized', { status: 401 });
  }
  
  return context.next();
};
```
