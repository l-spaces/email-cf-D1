# 分页功能更新说明

## 更新内容

为账号列表添加了完整的分页功能，包括前端UI和后端API支持。

### 后端API更新

#### 新增查询参数
- `page` - 页码（默认：1）
- `limit` - 每页数量（默认：20，最小：1，最大：100）
- `search` - 搜索关键词（匹配邮箱地址或备注）

#### 响应格式变化
添加了 `pagination` 对象：
```json
{
  "success": true,
  "data": [...],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 100,
    "totalPages": 5
  }
}
```

#### API示例
```bash
# 获取第一页，每页20条
curl http://localhost:8788/api/emails?page=1&limit=20

# 搜索
curl http://localhost:8788/api/emails?search=gmail

# 组合使用
curl "http://localhost:8788/api/emails?page=2&limit=50&search=test"
```

### 前端UI更新

#### 新增功能
1. **分页控件** - 包含首页、上一页、页码、下一页、末页按钮
2. **每页数量选择** - 支持10、20、50、100条/页
3. **分页信息显示** - 显示当前页范围和总数
4. **搜索与分页联动** - 搜索时自动跳转到第一页
5. **响应式设计** - 在移动设备上优化显示

#### UI组件
- 分页导航栏位于表格下方
- 左侧显示数据范围："第 1-20 条，共 100 条"
- 中间显示页码按钮（最多显示5个）
- 右侧显示每页数量选择器

#### 交互优化
- 第一页时禁用首页和上一页按钮
- 最后一页时禁用下一页和末页按钮
- 当前页按钮高亮显示
- 添加/删除账号后自动刷新当前页

### 性能优化

1. **数据库查询优化** - 使用LIMIT和OFFSET减少数据传输
2. **并行查询** - 同时查询数据和总数，减少响应时间
3. **搜索优化** - 使用LIKE查询支持模糊搜索
4. **前端优化** - 只渲染当前页数据，减少DOM操作

### 兼容性说明

- 向后兼容：不传分页参数时使用默认值（第1页，每页20条）
- API响应总是包含`pagination`对象，前端可根据是否存在该字段判断版本
- 搜索功能为可选，不影响现有调用方式

### 后续优化建议

1. 添加"跳转到指定页"输入框
2. 添加分页加载动画
3. 记住用户的每页数量偏好
4. 添加键盘快捷键支持（如PageUp/PageDown）
5. 考虑虚拟滚动代替分页（适合大数据量）

### 注意事项

- 搜索参数会在后端进行SQL LIKE查询，可能影响性能
- 建议在生产环境为数据库添加索引优化搜索性能
- 大数据量场景下建议限制最大每页数量
